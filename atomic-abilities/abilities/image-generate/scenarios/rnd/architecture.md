# 配方 · 结构化关系图（架构图 / 流程图 / 时序图）

method: code —— 写代码渲染，**不要**用栅格图像模型画结构化图（文字与框线会失真）。

## 工具选择（按可得性降级）
1. 首选 **mermaid**：写 `.mmd`，用 `mmdc`（@mermaid-js/mermaid-cli）渲染成 PNG；
2. 无 mmdc 时用 **graphviz**：写 `.dot`，`dot -Tpng`；
3. 都没有则**手写 SVG** 再转 PNG（或直接交付 SVG）。
把可复现源码（`.mmd` / `.dot` / `.svg`）和成品图一起存到产物目录。
环境缺工具时先尝试本地已装的；**不要联网安装依赖**，装不上就降级到 SVG 手写。

## 图型对应
- 架构图：`graph TD`/`flowchart TB`，分层自上而下；同层用 `subgraph` 归组；
- 流程图：`flowchart`，判定用菱形 `{}`，分支边标 `是/否`；
- 时序图：`sequenceDiagram`，participant 一行，箭头 `->>` 同步、`-->>` 返回。

## 布局与可读性规范
- 方向稳定（架构/流程 TB，时序 LR）；同类节点尺寸一致；
- **避免箭头穿过无关节点**：交叉多时用 subgraph 分组或调整顺序；
- 边尽量少标注；标注就一两词；
- 颜色 ≤2 组语义色，其余中性；用色表达含义时加一行图例；
- 文案**句子式**（不全大写、不标题式），中文图用中文标签；
- 节点文字简短，长说明放节点下方或省略。

## 输出
- 渲染为 PNG 存产物目录；文件名用场景+序号（如 `architecture-1.png`）；
- 若同时产出 SVG/源码，一并存，但 JSON 的 `images[].path` 指向 PNG（或 SVG）成品；
- `revised_prompt` 写明你采用的图型与渲染工具（如「mermaid flowchart TB，分层架构」）。
